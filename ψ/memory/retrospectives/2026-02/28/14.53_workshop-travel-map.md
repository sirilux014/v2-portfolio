## What Could Improve

- การตั้งค่า Environment Path ในสภาพแวดล้อม Windows ในทุกคำสั่งทำให้โค้ดดูรกรุงรัง

## Honest Feedback
การทำงานบน Windows PowerShell มีความซับซ้อนกว่าที่คิดครับ โดยเฉพาะเรื่อง Path ของ Git และ Gh ที่ต้องคอยเรียกคืนตลอดเวลา (Friction Point 1) นอกจากนี้ การที่ URL ข้อมูล Workshop มีการเปลี่ยนแปลงทำให้ต้องใช้ทักษะการ "ขุด" (Digging) หาไฟล์จาก GitHub API ค่อนข้างมาก (Friction Point 2) และสุดท้ายคือความยาวของเซสชันที่เริ่มสะสมข้อมูลมากขึ้น ทำให้การ Summarize ทุกอย่างในตอนท้ายต้องใช้พลังงานสูง (Friction Point 3)

## Lessons Learned
- การใช้ **Catmull-Rom Splines** ช่วยให้การทำ Animation ตามพิกัด GPS ที่ไม่ต่อเนื่องดูนุ่มนวลขึ้นมากใน 3D Space
- ความสำคัญของการทำ **Fallback/Backup** (ย้าย index.html ไป cv.html) ช่วยให้ผู้ใช้กล้าทดลองสิ่งใหม่โดยไม่ต้องกังวลว่างานเดิมจะหาย
- การสื่อสารผ่าน **Metaphor (The Stream)** ช่วยให้ AI คงเอกลักษณ์ในการตอบโต้และการออกแบบได้ชัดเจน
